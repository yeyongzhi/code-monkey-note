# Git基础知识
## Git 提交文案规范
提交文案格式：`type: 提交文案`

- **feat**：新功能（feature）
- **fix**：修复 bug
- **docs**：文档更新、变更
- **style**：代码格式（代码格式调整（不影响逻辑，如空格、分号））
- **refactor**：重构（既不是新增功能，也不是修复 bug 的代码变更）
- **perf**：性能优化（提升代码执行效率）
- **test**：添加或修改测试
- **build**：构建系统或依赖更新（如 webpack、npm）
- **ci**：CI 配置或脚本变更
- **chore**：杂项（如更新 .gitignore）
- **revert**：回滚之前的提交
